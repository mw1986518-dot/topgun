# TopGun：用 AI 多智能体并发推演，做出更明智的决策

> 当面对复杂决策时，你是否也曾感到无从下手？这篇文章将介绍一个开源工具，帮助你用多视角思维框架进行深度推演。

---

## 我们面临的决策困境

作为产品经理、创业者或团队管理者，我们经常需要面对复杂决策：

- **产品方向选择**：A 功能还是 B 功能优先？
- **创业方向验证**：这个商业模式是否可行？
- **技术方案评估**：选哪个技术栈？自研还是外采？
- **资源分配决策**：有限资源投入哪个方向？

传统的决策方式往往存在这些**痛点**：

### 1. 单一视角的盲区

我们习惯从自己熟悉的角度看问题。技术人员倾向技术方案，运营倾向运营方案。这种「锤子效应」让我们错过其他可能性。

### 2. 缺乏系统性深度思考

忙碌的工作节奏让我们难以静下心来深入思考。很多决策停留在表面，没有挖掘根本假设和潜在风险。

### 3. 决策过程缺乏质疑

团队讨论时，要么没人提出反对意见（群体思维），要么反对意见零散无序，无法形成有效的交叉验证。

### 4. 难以转化为行动

讨论很热烈，结论很模糊。开会时感觉想清楚了，落地时发现不知道从何开始。

---

## TopGun 的解决方案

**TopGun（顶级思维）** 是一个开源的 AI 辅助决策工具，通过**多智能体并发推演**的方式帮助你做出更明智的决策。

核心思路：**模拟一个「认知议会」，让多个代表不同思维框架的 AI 智能体同时对同一议题进行分析、质疑、修正，最终达成共识或识别风险。**

---

## 核心功能

### 1. 议题澄清：帮你问对问题

很多时候，我们不知道自己不知道什么。TopGun 会通过对话式追问，帮助你：

- 厘清真正的目标
- 发现隐藏的约束条件
- 明确成功的可验证标准
- 识别潜在的失败模式

### 2. 多框架并发推演：真正的并行思考

TopGun 内置 8 种思维框架，包括：

| 框架       | 视角                     |
| ---------- | ------------------------ |
| 第一性原理 | 回归本质，剥离假设       |
| 反脆弱     | 从混乱中获益，非对称风险 |
| 系统思维   | 反馈回路，二阶效应       |
| 水平思维   | 跨界模式匹配             |
| 行为经济学 | 认知偏差，激励机制       |
| ...        | ...                      |

你选择 3-5 个框架后，它们会**真正并行地**分析你的问题（使用 tokio::spawn 实现），就像多个专家同时独立思考。

### 3. 结构化交叉质询：智能体相互挑战

推演不是终点。每个智能体分析完成后，会进入「质询阶段」：

- 智能体 A 审查智能体 B 的结论，提出异议
- 智能体 B 针对异议进行修正
- 版本迭代：v1.0 → v2.0 → ...

这个过程会持续，直到没有新的异议，或达到最大迭代次数。

### 4. 风险显式标注：不隐藏分歧

当智能体无法达成共识时，TopGun 会**显式列出「容忍的风险」**：

- 风险摘要
- 证据
- 临时接受理由
- 后续需要采取的行动

这让决策者清楚知道：哪些地方还有争议，风险在哪里。

### 5. 行动计划生成：从分析到执行

分析得再好，不能落地也是空谈。TopGun 会：

- 基于共识结果生成针对性问题
- 通过交互式问答细化方案
- 输出结构化的行动计划

## 使用场景

### 产品决策

> "我们的产品应该主打 B 端还是 C 端？"

选择第一性原理、系统思维、行为经济学框架，从底层逻辑、市场生态、用户行为等多个角度分析。

### 创业验证

> "这个商业模式是否可行？"

选择反脆弱、水平思维、风险分析框架，评估下行风险、跨界参考案例、识别致命假设。

### 技术选型

> "应该选择哪个技术方案？"

选择系统思维、第一性原理框架，从长期维护成本、团队能力匹配、技术演进趋势等角度分析。

### 个人决策

> "是否应该接受这个工作机会？"

选择行为经济学、系统思维框架，分析激励机制、职业发展路径、机会成本。

## 快速开始

### 安装

```bash
# 克隆仓库
git clone https://github.com/mw1986518-dot/topgun.git
cd topgun

# 安装依赖
npm install

# 启动开发模式
npm run tauri dev
```

### 配置 LLM

1. 点击左侧「本地配置」
2. 添加你的 LLM 供应商信息
3. 支持任何兼容 OpenAI API 格式的服务

### 基本使用

1. 输入你的问题或议题
2. 回答 AI 的澄清问题
3. 选择推荐的思维框架
4. 观察多智能体并发推演
5. 查看共识和风险提示
6. 生成行动计划

## 技术亮点

### 真正的并发

使用 Rust 的 tokio::spawn 实现真正的并行执行，不是简单的顺序调用。这意味着 5 个智能体可以同时向 LLM 发起请求，大幅缩短等待时间。

### 现代技术栈

- **前端**：React 19 + TypeScript + Tailwind CSS 4 + Vite
- **后端**：Tauri 2 (Rust)
- **测试**：Vitest + Playwright

### 本地优先

- 所有数据存储在本地
- API Key 不会上传
- 无需联网也能查看历史

### 跨平台

支持 Windows、macOS、Linux。
